# SSH-Buddy v0.5.0 Release Notes

v0.5.0 adds explicit public SSH key installation through system OpenSSH tooling.

## Highlights

- Add Install public key action for saved server profiles.
- Add Copy install command action for review and manual execution.
- Use system `ssh-copy-id` in the selected external terminal so password and host-key prompts stay interactive.
- Resolve public key paths from saved SSH key references as `<private-key-path>.pub`.
- Support ProxyJump during public key install with `-o ProxyJump=<value>`.
- Add diagnostics for missing `.pub` files, missing `ssh-copy-id`, selected terminal, target user/host/port, ProxyJump, command preview, and backend spawn result.

## Security Model

- SSH-Buddy stores SSH key paths only.
- Private key contents are never read, imported, stored, copied, or sent to the remote server.
- Public key contents are not stored in the database.
- SSH-Buddy does not store server passwords, SSH key passphrases, sudo passwords, or remote-access passwords.
- Public key install requires explicit confirmation before launch.
- Commands are backend-built argv arrays with no shell interpolation.
- Interactive password and host-key prompts remain in the external terminal/OpenSSH flow.

## Manual Smoke Results

Validated on CachyOS/KDE Wayland:

- Copy install command works.
- Install public key launches `ssh-copy-id` in the selected external terminal.
- Password prompt appears interactively in the terminal.
- Public key installs successfully.
- SSH login works afterward using the selected key.
- Missing `.pub` file shows a clear recovery command.
- Missing `ssh-copy-id` shows a clear preflight error.
- Command preview never includes passwords.
- SSH-Buddy never reads, stores, or copies private key contents.

## Install Notes

Linux users should start with the AppImage:

```sh
chmod +x SSH-Buddy-v0.5.0-linux-amd64.AppImage
./SSH-Buddy-v0.5.0-linux-amd64.AppImage
```

On some Wayland/WebKitGTK sessions, use:

```sh
WEBKIT_DISABLE_DMABUF_RENDERER=1 ./SSH-Buddy-v0.5.0-linux-amd64.AppImage
```

SSH, SFTP, and public key install require OpenSSH clients in `PATH`: `ssh`, `sftp`, and `ssh-copy-id`.

SSH, SFTP, public key install, RDP, and tunnel launch require at least one supported external terminal in `PATH`: Konsole, kitty, Alacritty, WezTerm, GNOME Terminal, or xterm.

RDP launch requires FreeRDP in `PATH`: `xfreerdp3` or `xfreerdp`. RDP scaling support can vary by FreeRDP version and desktop environment.

Windows and macOS builds are unsigned for now. Windows SmartScreen, macOS Gatekeeper, and browser download warnings may appear.

## Known Limitations

- Linux remains the primary development target.
- Windows and macOS artifacts need broader real-machine testing.
- Public key install requires an existing `.pub` file next to the referenced private key.
- Public key install requires `ssh-copy-id`; no shell fallback is implemented.
- SSH-Buddy does not store passwords or key passphrases, so remote password prompts happen in the external terminal.
- SSH-Buddy does not deploy private keys, modify sudoers, automate sudo/root access, or silently install keys.
- RDP is still an external launcher, not an embedded desktop viewer.
- FreeRDP multi-monitor may require contiguous monitor layouts; use `xfreerdp3 /monitor-list` and try explicit monitor IDs.
- No VNC, sync, KeePassXC integration, embedded terminal, embedded RDP, or password vaulting.
