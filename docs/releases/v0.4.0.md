# SSH-Buddy v0.4.0 Release Notes

v0.4.0 adds RDP display/scaling options for high-DPI and resize-friendly FreeRDP sessions.

## Highlights

- Add RDP native/default scaling mode.
- Add RDP percentage scaling using verified FreeRDP values: `/scale:100`, `/scale:140`, and `/scale:180`.
- Add RDP smart sizing through `/smart-sizing`.
- Add RDP dynamic resolution through `+dynamic-resolution`.
- Show RDP scaling mode, scaling percent, smart sizing, dynamic resolution, fullscreen, width, and height in launch diagnostics.
- Document high-DPI RDP usage and FreeRDP/KDE troubleshooting.

## Security Model

- SSH-Buddy still does not store SSH, SFTP, RDP, sudo, or remote passwords.
- RDP launch never generates `/p:<password>`.
- RDP display/scaling options are allowlisted FreeRDP flags, not arbitrary command strings.
- Launch commands remain backend-built argv arrays with no shell interpolation.
- RDP still launches through the selected external terminal so FreeRDP can prompt for certificate trust and credentials.

## Manual Smoke Results

Validated on CachyOS/KDE Wayland:

- Native/default RDP works.
- Scale percentage works.
- Smart sizing works.
- Dynamic resolution works.
- Copy RDP command works and includes the expected scaling flags.
- RDP command preview never includes a password.
- RDP still launches through the external terminal.
- Certificate mode Trust on first use still works.
- Multi-monitor remains documented as a FreeRDP/KDE monitor-layout issue and is not a blocker.

## Install Notes

Linux users should start with the AppImage:

```sh
chmod +x SSH-Buddy-v0.4.0-linux-amd64.AppImage
./SSH-Buddy-v0.4.0-linux-amd64.AppImage
```

On some Wayland/WebKitGTK sessions, use:

```sh
WEBKIT_DISABLE_DMABUF_RENDERER=1 ./SSH-Buddy-v0.4.0-linux-amd64.AppImage
```

SSH and SFTP launch require OpenSSH clients in `PATH`: `ssh` and `sftp`.

SSH, SFTP, RDP, and tunnel launch require at least one supported external terminal in `PATH`: Konsole, kitty, Alacritty, WezTerm, GNOME Terminal, or xterm.

RDP launch requires FreeRDP in `PATH`: `xfreerdp3` or `xfreerdp`. RDP scaling support can vary by FreeRDP version and desktop environment.

Windows and macOS builds are unsigned for now. Windows SmartScreen, macOS Gatekeeper, and browser download warnings may appear.

## Known Limitations

- Linux remains the primary development target.
- Windows and macOS artifacts need broader real-machine testing.
- RDP is still an external launcher, not an embedded desktop viewer.
- FreeRDP multi-monitor may require contiguous monitor layouts; use `xfreerdp3 /monitor-list` and try explicit monitor IDs.
- Scaling behavior depends on the installed FreeRDP client and the local desktop environment.
- No RDP password storage, embedded RDP, VNC, SSH key deploy, sync, KeePassXC integration, or arbitrary FreeRDP option strings.
