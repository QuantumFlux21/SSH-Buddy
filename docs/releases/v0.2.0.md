# SSH-Buddy v0.2.0 Release Notes

v0.2.0 adds ProxyJump/bastion support and saved local SSH tunnel profiles while keeping SSH-Buddy focused on OpenSSH-backed connection management.

## Highlights

- Add optional ProxyJump values to saved server profiles.
- Launch SSH sessions with OpenSSH `-J` when ProxyJump is configured.
- Preserve detected `ProxyJump` values during selected `~/.ssh/config` import.
- Add saved local tunnel profiles per server.
- Copy tunnel commands built as `ssh -N -L ...`.
- Launch tunnels in an external terminal through the existing terminal launcher.
- Keep remote forwarding, SOCKS forwarding, SFTP, RDP, embedded terminals, sync, password storage, and KeePassXC out of scope.

## Security Model

- SSH-Buddy still uses system OpenSSH instead of implementing SSH crypto.
- Private key contents, SSH passwords, passphrases, sudo passwords, and remote passwords are not stored.
- SSH and tunnel launch use argv/process APIs without shell interpolation.
- ProxyJump and tunnel values are validated before save and launch.
- Agent forwarding is not enabled by default.
- Sudo/root automation is not implemented.
- Local tunnel profiles default to `127.0.0.1`; binding to wider interfaces can expose forwarded services and should be intentional.

## Expected Release Artifacts

The release workflow builds platform-specific files:

- Linux: AppImage
- Windows: NSIS `.exe` installer
- macOS Apple Silicon: `.dmg`
- macOS Intel: `.dmg`

There is one app and one codebase, but not one universal installer for every operating system.

## Install Notes

Linux users should start with the AppImage:

```sh
chmod +x SSH-Buddy-v0.2.0-linux-amd64.AppImage
./SSH-Buddy-v0.2.0-linux-amd64.AppImage
```

On some Wayland/WebKitGTK sessions, use:

```sh
WEBKIT_DISABLE_DMABUF_RENDERER=1 ./SSH-Buddy-v0.2.0-linux-amd64.AppImage
```

SSH and tunnel launch require the OpenSSH client and at least one supported external terminal in `PATH`: Konsole, kitty, Alacritty, WezTerm, GNOME Terminal, or xterm.

Windows and macOS builds are unsigned for now. Windows SmartScreen, macOS Gatekeeper, and browser download warnings may appear.

## Known Limitations

- Linux remains the primary development target.
- Windows and macOS artifacts need broader real-machine testing.
- Local AppImage packaging can fail on rolling-release Linux when `linuxdeploy` strips newer system libraries with `.relr.dyn` sections; use the GitHub Actions Ubuntu release workflow for release AppImages.
- Remote forwarding with `-R` is not implemented.
- SOCKS forwarding with `-D` is not implemented.
- SSH config import is read-only and preview-first.
- No SFTP, RDP, embedded terminal, sync, KeePassXC integration, password storage, or sudo/root automation.

## Manual Smoke Tests

Before publishing the GitHub release:

1. Create, edit, favorite, and delete a server profile.
2. Add, edit, remove, and launch a ProxyJump-backed SSH profile.
3. Import a host with `ProxyJump` from `~/.ssh/config` and confirm the saved profile keeps it.
4. Add, edit, copy, launch, and delete a local tunnel profile.
5. Delete a server with tunnels and confirm associated tunnel profiles are removed.
6. Confirm invalid ProxyJump values, invalid tunnel ports, invalid host values, missing `ssh`, and missing terminal cases show clear errors.
7. Smoke-test the Linux AppImage, including the Wayland workaround if needed.
