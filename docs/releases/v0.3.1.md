# SSH-Buddy v0.3.1 Release Notes

v0.3.1 is a focused bugfix release for Linux desktop usability after v0.3.0.

## Highlights

- Fix Tauri desktop clipboard copy behavior with the official clipboard manager plugin.
- Add a manual-copy fallback when clipboard access fails.
- Improve SSH, SFTP, tunnel, and RDP launch diagnostics.
- Clarify launch messages so a spawned external process is not reported as a confirmed remote connection.
- Launch FreeRDP through the selected external terminal so certificate trust and password prompts have a usable TTY.
- Add RDP certificate mode support: default/prompt, `/cert:tofu`, and `/cert:ignore`.
- Document FreeRDP/KDE multi-monitor troubleshooting, including monitor layout gaps and `xfreerdp3 /monitor-list`.

## Security Model

- SSH-Buddy still does not store SSH, SFTP, RDP, sudo, or remote passwords.
- RDP launch never generates `/p:<password>`.
- Clipboard copy and launch actions do not change private key handling.
- Launch commands remain backend-built argv arrays with no shell interpolation.
- RDP certificate handling is allowlisted. `/cert:ignore` is available but marked less secure and is never selected silently.

## Manual Smoke Results

Validated on CachyOS/KDE Wayland:

- Copy SSH command works.
- Copy SFTP command works.
- Open SSH works with Alacritty.
- Open SFTP works with Alacritty.
- RDP works when launched through the external terminal.
- RDP trust-on-first-use certificate mode works.
- FreeRDP prompts for the password correctly in the terminal.
- Multi-monitor RDP may still fail on KDE monitor layouts with FreeRDP monitor gaps; this is documented troubleshooting, not a v0.3.1 blocker.

## Install Notes

Linux users should start with the AppImage:

```sh
chmod +x SSH-Buddy-v0.3.1-linux-amd64.AppImage
./SSH-Buddy-v0.3.1-linux-amd64.AppImage
```

On some Wayland/WebKitGTK sessions, use:

```sh
WEBKIT_DISABLE_DMABUF_RENDERER=1 ./SSH-Buddy-v0.3.1-linux-amd64.AppImage
```

SSH and SFTP launch require OpenSSH clients in `PATH`: `ssh` and `sftp`.

SSH, SFTP, RDP, and tunnel launch require at least one supported external terminal in `PATH`: Konsole, kitty, Alacritty, WezTerm, GNOME Terminal, or xterm.

RDP launch requires FreeRDP in `PATH`: `xfreerdp3` or `xfreerdp`.

Windows and macOS builds are unsigned for now. Windows SmartScreen, macOS Gatekeeper, and browser download warnings may appear.

## Known Limitations

- Linux remains the primary development target.
- Windows and macOS artifacts need broader real-machine testing.
- RDP is still an external launcher, not an embedded desktop viewer.
- FreeRDP multi-monitor may require contiguous monitor layouts; use `xfreerdp3 /monitor-list` and try explicit monitor IDs.
- No RDP password storage, embedded RDP, VNC, SSH key deploy, sync, KeePassXC integration, or arbitrary FreeRDP option strings.
