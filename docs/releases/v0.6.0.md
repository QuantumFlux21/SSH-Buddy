# SSH-Buddy v0.6.0 Release Notes

v0.6.0 improves KDE/Konsole launch reliability and adds manual selected-server reachability checks.

## Highlights

- Fix Konsole launch behavior by using `konsole --noclose -e ...`.
- Add Settings -> Test terminal for validating the selected external terminal.
- Add server status dots in the left server list.
- Add selected-server reachability panel with ping and primary TCP port details.
- Add online/degraded/offline/unknown status model based on ping and TCP results.
- Add ping latency sparkline using recent in-memory samples.
- Add manual selected-host-only port scan for a small allowlist.
- Add full spawned argv preview to launch diagnostics.

## Status Model

- Unknown: no check has run yet.
- Online: ping and the primary TCP port both respond.
- Degraded: ping works but the primary TCP port fails, or ping fails/is unavailable while TCP works.
- Offline: both ping and the primary TCP port fail.
- Checking: a manual check is currently running.

The primary TCP port is the SSH profile port by default. If RDP is enabled for the selected profile, SSH-Buddy checks the RDP port as the primary service port. Ping may be blocked by firewalls or unavailable in sandboxed environments, so SSH-Buddy always runs the TCP connect check as well.

## Manual Port Scan

The Scan ports action is manual and selected-host only. It checks only this allowlist:

- 22 SSH
- 80 HTTP
- 443 HTTPS
- 3389 RDP
- 5432 PostgreSQL
- 6379 Redis
- 8080 HTTP-alt
- 8443 HTTPS-alt

SSH-Buddy does not scan subnets, expand ranges, run background discovery, or store long-term monitoring history. Use this only for servers you own or administer.

## Security Model

- SSH-Buddy still does not store SSH, SFTP, RDP, sudo, or remote passwords.
- Private key contents are never read, imported, copied, or stored.
- Terminal launch, ping checks, TCP checks, and selected-host port scans use backend-controlled argv/process calls with no shell interpolation.
- Port scanning is selected-host only and allowlisted.
- No subnet discovery, background network-wide scanning, password vaulting, or embedded terminal was added.

## Manual Smoke Results

- Konsole opens and stays open using `--noclose`.
- Test terminal works.
- Open SSH works with Konsole.
- Open SFTP works with Konsole.
- Alacritty still works.
- Last launch attempt shows the full argv preview.
- Left server list status dots update correctly.
- Selected server reachability panel shows ping/TCP details.
- Ping latency sparkline works.
- Manual Scan ports checks only the selected server.
- Port scan uses only the allowlisted ports.
- No broad/subnet scanning was added.

## Install Notes

Linux users should start with the AppImage:

```sh
chmod +x SSH-Buddy-v0.6.0-linux-amd64.AppImage
./SSH-Buddy-v0.6.0-linux-amd64.AppImage
```

On some Wayland/WebKitGTK sessions, use:

```sh
WEBKIT_DISABLE_DMABUF_RENDERER=1 ./SSH-Buddy-v0.6.0-linux-amd64.AppImage
```

SSH, SFTP, and public key install require OpenSSH clients in `PATH`: `ssh`, `sftp`, and `ssh-copy-id`.

SSH, SFTP, public key install, RDP, tunnel launch, and terminal test require at least one supported external terminal in `PATH`: Konsole, kitty, Alacritty, WezTerm, GNOME Terminal, or xterm.

RDP launch requires FreeRDP in `PATH`: `xfreerdp3` or `xfreerdp`.

Windows and macOS builds are unsigned for now. Windows SmartScreen, macOS Gatekeeper, and browser download warnings may appear.

## Known Limitations

- Linux remains the primary development target.
- Windows and macOS artifacts need broader real-machine testing.
- Reachability history is in memory only and is not long-term monitoring.
- Ping may be blocked by firewalls or unavailable in some environments.
- Port scan is intentionally limited to the selected host and built-in allowlist.
- No VNC, sync, KeePassXC integration, embedded terminal, embedded RDP, background scanning, or password vaulting.
