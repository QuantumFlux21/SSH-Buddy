# SSH-Buddy v0.1.0 Release Notes

This is the first public release of SSH-Buddy, a Linux-first desktop SSH manager for homelab users.

## Highlights

- Local SQLite persistence for app metadata.
- Server profiles with hostname/IP, port, username, identity key reference, group, tags, favorite state, and plaintext notes.
- Group, tag, and SSH key reference management.
- External terminal SSH launch through the system `ssh` command.
- Copy SSH command action.
- Web/admin links per server with safe URL validation.
- Read-only `~/.ssh/config` import preview and selected import.
- Dark Tauri/React desktop UI.

## Security Model

- SSH-Buddy uses system OpenSSH and does not implement SSH cryptography.
- Private key contents are not read into the database.
- SSH passwords, passphrases, sudo passwords, and remote passwords are not stored.
- Sudo/root automation is not implemented.
- SSH launch uses argv/process APIs instead of shell interpolation.
- Web/admin links allow only `http://` and `https://` URLs and reject embedded credentials.

## Expected Artifacts

- Linux: `.AppImage`
- Windows: NSIS `.exe` installer
- macOS: `.dmg` for x64 and Apple Silicon

This is one codebase with multiple platform-specific release files. There is no single universal installer for every operating system.

## Install Notes

Linux users should start with the AppImage. The app launches SSH sessions in an external terminal and expects:

- OpenSSH client available as `ssh`
- One supported terminal in `PATH`: Konsole, kitty, Alacritty, WezTerm, GNOME Terminal, or xterm

On Linux, run the AppImage directly after making it executable:

```sh
chmod +x SSH-Buddy-v0.1.0-linux-amd64.AppImage
./SSH-Buddy-v0.1.0-linux-amd64.AppImage
```

On some Wayland/WebKitGTK sessions, development builds or the AppImage may need:

```sh
WEBKIT_DISABLE_DMABUF_RENDERER=1 npm run tauri dev
WEBKIT_DISABLE_DMABUF_RENDERER=1 ./SSH-Buddy-v0.1.0-linux-amd64.AppImage
```

## Known Limitations

- This is a pre-1.0 release and has not had broad real-machine testing across all target platforms.
- Windows and macOS builds are unsigned and may trigger OS warnings.
- macOS builds are not notarized.
- SSH config import is preview-first and only imports selected concrete `Host` aliases.
- SSH config import does not edit `~/.ssh/config`.
- ProxyJump is detected in preview but not yet stored or used for launch.
- SFTP, RDP, tunnels, embedded terminals, sync, KeePassXC, and privileged command automation are out of scope for this release.
