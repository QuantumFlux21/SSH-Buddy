# SSH-Buddy v0.3.0 Release Notes

v0.3.0 adds external SFTP and RDP launch actions while keeping SSH-Buddy focused on system tools, saved local metadata, and password-free app storage.

## Highlights

- Launch SFTP in an external terminal using the system OpenSSH `sftp` client.
- Copy SFTP commands generated from saved server profile data.
- Include SFTP username, host, port, identity file reference, and ProxyJump values.
- Launch RDP externally through FreeRDP, preferring `xfreerdp3` and falling back to `xfreerdp`.
- Copy RDP commands generated from saved RDP settings.
- Store optional RDP username, domain, port, fullscreen, multi-monitor, monitor IDs, dimensions, and color depth.
- Pass FreeRDP multi-monitor settings as `/multimon` and validated monitor selections such as `/monitors:0,1`.
- Keep embedded SFTP/RDP, password storage, KeePassXC, sync, and arbitrary FreeRDP option strings out of scope.

## Security Model

- SSH-Buddy still delegates SSH/SFTP behavior to system OpenSSH.
- SSH-Buddy delegates RDP behavior to FreeRDP.
- Private key contents, SSH passwords, SFTP passwords, RDP passwords, passphrases, sudo passwords, and remote passwords are not stored.
- SFTP launches through backend-owned argv/process APIs without shell interpolation.
- RDP launches through backend-owned argv/process APIs without shell interpolation.
- RDP launch never includes `/p:` password arguments.
- RDP monitor IDs are validated as comma-separated monitor numbers and are not arbitrary FreeRDP options.
- Agent forwarding is not enabled by default.
- Sudo/root automation is not implemented.

## Expected Release Artifacts

The release workflow builds platform-specific files:

- Linux: AppImage
- Windows: NSIS `.exe` installer
- macOS Apple Silicon: `.dmg`
- macOS Intel: `.dmg`

There is one app and one codebase, but not one universal installer for every operating system.

## Install Notes

Linux users should start with the AppImage:

```sh
chmod +x SSH-Buddy-v0.3.0-linux-amd64.AppImage
./SSH-Buddy-v0.3.0-linux-amd64.AppImage
```

On some Wayland/WebKitGTK sessions, use:

```sh
WEBKIT_DISABLE_DMABUF_RENDERER=1 ./SSH-Buddy-v0.3.0-linux-amd64.AppImage
```

SSH and SFTP launch require OpenSSH clients in `PATH`: `ssh` and `sftp`.

SSH, SFTP, and tunnel launch require at least one supported external terminal in `PATH`: Konsole, kitty, Alacritty, WezTerm, GNOME Terminal, or xterm.

RDP launch requires FreeRDP in `PATH`: `xfreerdp3` or `xfreerdp`.

Windows and macOS builds are unsigned for now. Windows SmartScreen, macOS Gatekeeper, and browser download warnings may appear.

## Known Limitations

- Linux remains the primary development target.
- Windows and macOS artifacts need broader real-machine testing.
- Local AppImage packaging can fail on rolling-release Linux when `linuxdeploy` strips newer system libraries with `.relr.dyn` sections; use the GitHub Actions Ubuntu release workflow for release AppImages.
- SFTP is an external launcher, not an embedded file browser.
- RDP is an external launcher, not an embedded desktop viewer.
- RDP credentials are handled interactively by FreeRDP only.
- SSH config import is read-only and preview-first.
- Remote forwarding with `-R` is not implemented.
- SOCKS forwarding with `-D` is not implemented.
- No embedded terminal, sync, KeePassXC integration, password storage, or sudo/root automation.

## Manual Smoke Tests

Before publishing the GitHub release:

1. Create, edit, favorite, and delete a server profile.
2. Launch SSH for a saved server profile.
3. Launch SFTP for a saved server profile.
4. Copy an SFTP command and confirm it includes port, identity file, and ProxyJump values when configured.
5. Configure RDP settings with username, domain, port, dimensions, color depth, fullscreen, and multi-monitor options.
6. Copy an RDP command and confirm it uses `xfreerdp3` or `xfreerdp` with expected argv-style options.
7. Configure multi-monitor and monitor IDs, then confirm the command includes `/multimon` and `/monitors:0,1`.
8. Launch RDP and confirm FreeRDP handles credential prompts interactively.
9. Confirm invalid RDP ports, dimensions, color depths, monitor IDs, and monitor IDs without multi-monitor enabled show clear errors.
10. Confirm missing `sftp`, missing FreeRDP, missing selected key file, and missing supported terminal cases show clear errors.
11. Smoke-test the Linux AppImage, including the Wayland workaround if needed.
